# 1. Install terraform
# 1.1. Preparation to terraform installation

```
#creating a new project
source /opt/stack/devstack/openrc admin admin
openstack project create cluster
openstack role add --project cluster --user admin admin
source /opt/stack/devstack/openrc admin cluster
#installation
mkdir terraform
cd terraform
curl -o terraform.zip https://releases.hashicorp.com/terraform/1.14.3/terraform_1.14.3_linux_amd64.zip
unzip terraform.zip
sudo mv terraform /usr/local/bin

```

# 1.2. Validation

```
terraform -version
terraform validate
```

# 1.3. Init terraform


added provider to main.tf
`terraform init`
edited provider in main.tf
`terraform init -upgrade`


# 1.4. Creating tf files and provisioning k8s cluster

`openstack keypair create --type ssh clusterkeys > cluster`

# 2. k8s installation and validation
# 2.1 Installation

The script bellow is passed to a VM using terraform and cloud init.  
It installs kubeadm, kublet, kubctl and initiates a k8s cluster.  
The script also generates a post init script that needs to be run to finalize initialization: [post-init.sh](#23-finalize-cluster-initialization)  
[k8s install](scripts/init.sh)


# 2.2 Health check

```
#some commands for checking health
kubectl get nodes
kubectl get componentstatuses
kubectl get pods -n kube-system
kubectl get --raw='/readyz?verbose'
kubectl cluster-info
```
# 2.3 Finalize cluster initialization

post-init.sh is generated by the installation script inside the vm and should be run as ubuntu user
```
#post-init.sh
#copy kubectl config to accessible folder
mkdir -p $HOME/.kube
sudo cp /etc/kubernetes/admin.conf $HOME/.kube/config
sudo chown $(id -u):$(id -g) $HOME/.kube/config
#apply CNI
kubectl apply -f calico.yaml
#restart
sudo systemctl restart kubelet
```

```
#test with a simple pod
kubectl run smoke-test --image=busybox --restart=Never -- sleep 10
kubectl get pod smoke-test
```

# 3. Test deployment using k8s

Tutorial: https://kubernetes.io/docs/tutorials/kubernetes-basics/deploy-app/deploy-intro/

```
#create deployment
kubectl create deployment kubernetes-bootcamp --image=gcr.io/google-samples/kubernetes-bootcamp:v1
#check the deployment, it takes time for it to get ready
kubectl get deployments
#open a proxy server in another window
kubectl proxy
#in the first window connect to the api 
curl http://localhost:8001/version
#export the pod name
export POD_NAME=$(kubectl get pods -o go-template --template '{{range .items}}{{.metadata.name}}{{"\n"}}{{end}}')
echo Name of the Pod: $POD_NAME
#when the pod is ready access the pod
curl http://localhost:8001/api/v1/namespaces/default/pods/$POD_NAME:8080/proxy/

```
